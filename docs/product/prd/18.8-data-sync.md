# PRD: Data & Sync (Encrypted)

> **ID:** 18.8 | **Status:** Shipped | **Priority:** P0
> **Last Updated:** 2026-04-04

---

## Purpose

Provide encrypted local storage and multi-backend sync (iCloud + Supabase) with zero-knowledge architecture — ensuring user health data never leaves the device unencrypted.

## Business Objective

Privacy-first architecture is FitMe's primary differentiator. Zero-knowledge encryption and multi-backend sync enable the "privacy-first fitness" positioning that no competitor matches. This is a critical trust signal for health-conscious users.

## Target Persona(s)

- **All personas** — data storage is infrastructure
- **Primary differentiator for:** The Data-Driven Optimizer (cares deeply about data privacy)

## Current Implementation

### Encryption
| Layer | Technology | Details |
|-------|------------|---------|
| At-rest encryption | AES-256-GCM | All personal data encrypted before storage |
| Key management | Secure Enclave (P-256) | Biometric ACL, keys never leave device |
| Additional cipher | ChaCha20-Poly1305 | Via CryptoKit for specific data types |
| Sync format | `.ftenc` blobs | Encrypted binary — servers store opaque blobs |

### Sync Backends
| Backend | Protocol | Status |
|---------|----------|--------|
| CloudKit (iCloud Private DB) | Native CloudKit | Shipped |
| Supabase (PostgreSQL + Realtime) | REST + Realtime subscriptions | Shipped |

### Architecture
- **EncryptedDataStore** — main data access layer, all reads/writes go through encryption
- **CloudKitSyncService** — iCloud sync with conflict resolution
- **SupabaseSyncService** — Supabase sync with encrypted payload push/pull
- **Zero-knowledge guarantee** — servers only receive `.ftenc` blobs, cannot decrypt

### Key Features
- AES-256-GCM encryption for all personal data
- Secure Enclave key storage with biometric ACL
- Dual sync backends (user choice in Settings)
- Incremental sync (only changed records)
- Conflict resolution (last-write-wins with timestamps)
- Offline-first (all features work without network)
- Cloud metadata tracking (needsSync, cloudRecordID, lastModified)
- GDPR data export (JSON format via share sheet)
- GDPR data deletion (10-step cascade across 9 stores)

### Data Stores (9 total)
DailyLog, ExerciseLog, CardioLog, NutritionLog, SupplementLog, DailyBiometrics, MealTemplate, UserPreferences, AIRecommendation
