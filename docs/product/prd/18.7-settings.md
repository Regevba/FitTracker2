# PRD: Settings

> **ID:** 18.7 | **Status:** Shipped | **Priority:** P1
> **Last Updated:** 2026-04-04

---

## Purpose

Provide a grouped settings architecture for managing account, preferences, data & sync, privacy, and app configuration.

## Business Objective

Settings support user control and trust — critical for a health data app. Key settings (analytics toggle, data export, account deletion) are GDPR compliance requirements. Well-organized settings reduce support burden.

## Target Persona(s)

- **All personas** — settings is a utility feature

## Current Implementation

### Settings Categories (5 Groups)
| Category | Options |
|----------|---------|
| Account & Security | Profile, Apple Sign In, passkey management, biometric lock, delete account |
| Data & Sync | Sync backend (iCloud/Supabase), export data, analytics toggle, data retention |
| Training | Zone 2 HR boundaries, rest timer defaults, day type preferences |
| Nutrition | Macro targets, meal slot names, supplement definitions |
| App | Theme, notifications, about/version |

### Key Features
- Grouped architecture (not a flat form)
- GDPR settings: analytics opt-in/out toggle, data export, account deletion
- Delete Account (AccountDeletionService — 10-step cascade, 30-day grace period)
- Export Data (DataExportService — JSON export via iOS share sheet)
- Biometric lock toggle
- Zone 2 HR threshold customization

### Integration Points
- ConsentManager — analytics consent toggle
- AccountDeletionService — delete account flow
- DataExportService — data export flow
- HealthKitService — HR zone configuration
