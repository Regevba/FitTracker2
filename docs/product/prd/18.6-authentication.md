# PRD: Authentication & Account

> **ID:** 18.6 | **Status:** Shipped | **Priority:** P0
> **Last Updated:** 2026-04-04

---

## Purpose

Provide secure, frictionless authentication with Apple-first identity management, supporting Apple Sign In, passkeys (WebAuthn), email/OTP, and biometric app lock — with zero-password architecture.

## Business Objective

Authentication is the gateway to the app. Friction here causes abandonment. Apple Sign In reduces signup to one tap. Passkeys eliminate password-related support burden. Biometric lock protects sensitive health data.

## Target Persona(s)

- **All personas** — every user must authenticate

## Current Implementation

### Auth Methods
| Method | Status | Details |
|--------|--------|---------|
| Apple Sign In | Shipped | Primary auth flow, ASAuthorizationController |
| Passkeys (WebAuthn) | Shipped | FIDO2/WebAuthn via ASAuthorizationPlatformPublicKeyCredentialProvider |
| Email/OTP | Shipped | Email + one-time password flow |
| Biometric Lock | Shipped | Face ID / Touch ID via LAContext for app re-entry |

### Screens
| Screen | Description |
|--------|-------------|
| Auth Hub | Trust-first design — login and create-account modes |
| AccountPanelView | Account settings — profile, sign out, delete account |

### Architecture
- **SignInService** — handles all auth flows (Apple, passkey, email)
- **AuthManager** — state machine managing auth lifecycle
- **Secure Enclave** — P-256 key storage with biometric ACL
- **Simulator auto-login** — DEBUG mode bypasses auth for development

### Key Features
- One-tap Apple Sign In (primary recommended flow)
- Passkey registration and authentication
- Email/OTP fallback
- Biometric app lock (Face ID/Touch ID)
- JWT-based session management
- Auto-login in simulator (DEBUG)
- Account deletion (GDPR Article 17) via AccountDeletionService
