# UX & UI Build Prompts

> Collection of all handoff prompts used to brief other agents (Figma MCP, SwiftUI implementation agents, audit agents) on UX and UI build work for FitMe. Folder layout split 2026-05-06 to separate UX (what-and-why) from UI/design (how-it-looks). Auto-generated by `/ux prompt` and `/design prompt` during PM workflow Phase 3.

---

## Folder layout (split established 2026-05-06)

| Folder | Owner | Output of | Purpose |
| --- | --- | --- | --- |
| [`ux/`](ux/) | `/ux` skill | `/ux prompt {feature}` | What-and-why prompts: user flows, principle application, state coverage, accessibility requirements |
| [`ui/`](ui/) | `/design` skill | `/design prompt {feature}` | How-it-looks prompts: token contracts, component contracts, motion contracts, Figma node plan |
| [`_legacy/`](_legacy/) | (historical) | hand-authored pre-automation | Iteration 1 / iteration 2 / Figma prototype prompts that pre-date the auto-generation contract |

Auto-generated prompts always come in pairs — `/ux prompt` writes to `ux/` and `/design prompt` writes to `ui/` with matching `{YYYY-MM-DD}-{feature}-` filename prefixes so the receiving agent can read them together. See `/ux prompt` in [`../skills/ux.md`](../skills/ux.md) and `/design prompt` in [`../skills/design.md`](../skills/design.md) for the generation contract.

## What goes here

This folder contains:

1. **Auto-generated UX prompts** under `ux/` — produced by `/ux prompt {feature}` once Phase 3 of the PM workflow is approved. Every new feature in Phase 3 onward generates a UX-build prompt here.
2. **Auto-generated design prompts** under `ui/` — produced by `/design prompt {feature}` for the same feature. Pairs with the UX prompt above.
3. **Legacy hand-authored prompts** under `_legacy/` — historical prompts used to build FitMe's UI before the auto-generation contract (Figma prototypes, iteration 2 screens, prototype audits, onboarding v2 pilot, UX foundations deep dive). Kept as reference material for the prompt-template authors and as audit context.

## Why this folder exists

Before 2026-04-08 these prompts were scattered across `docs/project/` and `docs/design-system/` with no clear naming or discoverability. After adding `/ux prompt` and `/design prompt` to the skills ecosystem, we needed a single place where:

- The auto-generated prompts could land deterministically
- The hand-authored historical prompts could be reviewed as references for the auto-generation templates
- The team could see what's been sent to other agents without hunting through multiple folders

The 2026-05-06 split into `ux/` + `ui/` + `_legacy/` formalizes the auto-generation pairing AND keeps the legacy prompts visually separate from the modern automated path.

## How to use a prompt from here

1. Open the prompt file (under `ux/` or `ui/` for current features; `_legacy/` for reference)
2. Read the "target agent" header to see which MCP / skill / Claude Console profile it's meant for
3. Copy the prompt body (everything below the metadata block)
4. Paste into the target agent's interface — OR — for design-build prompts, simply invoke `/design build {feature}` and the skill will use the saved prompt automatically (Figma MCP path) or surface it for manual handoff (fallback path)
5. Wait for the receiving agent to produce its output (Figma nodes, Swift code, screenshots)
6. Log the result back into the feature's `state.json` under `figma_node_ids` (for design builds) or `phases.ux_or_integration.handoff_results` (for UX builds)
