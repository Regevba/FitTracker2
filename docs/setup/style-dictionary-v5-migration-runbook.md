> **Status:** READY TO EXECUTE — needs a network-capable environment to `npm install` v5 + golden-verify. Drafted 2026-06-08 (the local agent env has no npm-registry access, so the build could not be run to verify here).
> **Tracks:** cadence item OP-2026-06-02-03 · Dependabot PR #668 (style-dictionary 3.9.2 → 5.4.3, recurring-BLOCKED) · the #577 incident (broke `make tokens` on main ~30 min) · #596 (partial migration: config refactored to v5 but dep left at v3 → CI errored `Unknown transforms`).

## The non-negotiable rule (the #577/#596 lesson)

This is a **coordinated 4-file change that MUST land in ONE PR**, gated on a golden-snapshot diff that proves the regenerated `DesignTokens.swift` is byte-identical (date line normalized) to the v3 output. Splitting config-vs-dependency is what broke main twice.

1. `package.json` — `style-dictionary` `^3.9.2 → ^5.4.3` + the `tokens` script
2. `package-lock.json` — regenerated by the install
3. `sd.config.js → sd.config.mjs` — CJS→ESM + the v5 API changes
4. `Makefile` `tokens:` target — the v5 invocation

## v3 → v5 API change map (apply to the config)

| v3 | v5 | Notes |
|---|---|---|
| `const SD = require('style-dictionary')` + `module.exports = {…}` | `import SD from 'style-dictionary'` + `export default {…}` | v5 is **ESM-only** → rename `sd.config.js` → `sd.config.mjs` (keeps `scripts/check-tokens.js` CJS untouched; do NOT add `"type":"module"`) |
| `matcher: (t) => …` | `filter: (t) => …` | transform predicate rename |
| `transformer: (t) => …` | `transform: (t) => …` | transform fn rename |
| `formatter: ({dictionary}) => …` | `format: ({dictionary}) => …` | format fn rename |
| `StyleDictionary.extend(cfg).buildAllPlatforms()` (sync) | `const sd = new StyleDictionary(cfg); await sd.buildAllPlatforms()` (async) | build is async in v5 |
| transform `name/cti/camel` | **`name/camel`** ⚠️ | **VERIFY** — the #596 error (`Unknown transforms "name/camel"`) confirms v5 renamed this. `attribute/cti` likewise needs checking against v5's built-in registry. |
| `dictionary.allTokens`, `token.value/.type/.path/.comment` | unchanged | v5 with `usesDtcg:false` (default) keeps the legacy `value`/`type` accessors — our `tokens.json` is legacy-format, so no `$value` change needed |

`color/swift` (custom transform) + `swift/fittracker-tokens` (custom format) port mechanically via the renames above — the **function bodies are unchanged** (same hex→RGB math, same line emission).

## Makefile target (v5)

```make
tokens: node_modules
	node_modules/.bin/style-dictionary build --config sd.config.mjs
	@echo "✅  DesignTokens.swift regenerated"
```
(The v5 CLI loads the ESM config + runs the module-level `registerTransform`/`registerFormat` side-effects before building. `npm run tokens` script → `style-dictionary build --config sd.config.mjs`.)

## Verify loop (the gate — do BEFORE committing)

```bash
cp FitTracker/DesignSystem/DesignTokens.swift /tmp/tokens.golden.swift   # v3 baseline
npm install -D style-dictionary@^5.4.3                                   # → updates package.json + lock
# … port sd.config.js → sd.config.mjs per the table; update Makefile + npm script …
make tokens
# date-normalized diff (matches scripts/check-tokens.js which strips the AUTO-GENERATED date line):
diff <(sed -E 's#// AUTO-GENERATED by .* [0-9-]{10}#DATE#' /tmp/tokens.golden.swift) \
     <(sed -E 's#// AUTO-GENERATED by .* [0-9-]{10}#DATE#' FitTracker/DesignSystem/DesignTokens.swift)
make tokens-check    # the CI gate — must pass
```
If the diff is empty and `make tokens-check` passes → the migration is byte-equivalent → safe to commit all 4 files + close Dependabot #668. If the diff shows token changes, the most likely culprit is a wrong built-in transform name (the ⚠️ row) — fix the transform list and re-run.
